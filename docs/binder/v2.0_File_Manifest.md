# Binder v2.0 File Manifest

This document defines the exact file-level implementation plan for Binder Subsystem v2.0.

It should be used when building the `binder/` directory in the `feat/v1-implementation` branch.

---

## 1. Directory Tree

```text
binder/
  ingestion/
    scan_sources.py
    manifest.py
  classify/
    classifier.py
    rules.py
  dedupe/
    hash_index.py
    near_duplicate.py
  index/
    build_master_index.py
    build_curated_indexes.py
  page_resolver/
    resolve_pages.py
    validate_refs.py
  assembly/
    build_full_binder.py
    build_disclosure_packet.py
    build_core_packet.py
  hearing_pack/
    build_hearing_pack.py
  decision_support/
    build_quick_packet.py
    build_decision_sheet.py
  schemas/
    source_manifest.py
    exhibit_index.py
```

- `scan_sources.py`

- `hash_index.py`

## 5. CLI Entry Points

### Scan + manifest
```bash
python -m binder.ingestion.scan_sources --root <path> --out <path>
```

### Master index build
```bash
python -m binder.index.build_master_index --manifest <csv> --out <path>
```

### Curated index build
```bash
python -m binder.index.build_curated_indexes --master <csv> --out <path>
```

## 7. Integration Points

### Worker integration
- binder pipeline can be triggered as a background job
- outputs stored in MinIO as artifacts

### API integration
- expose endpoints to trigger scan/index builds
- expose endpoints to fetch manifest and indexes

### Frontend integration
- binder workspace page
- index viewer
- duplicate review UI (later)

---

## 8. Build Order

Implement files in this sequence:

1. `scan_sources.py`
2. `manifest.py`
3. `rules.py`
4. `classifier.py`
5. `hash_index.py`
6. `near_duplicate.py`
7. `build_master_index.py`
8. `build_curated_indexes.py`

Everything else can be added incrementally after v2.0 baseline is working.

---

## 9. Definition of Done

Binder v2.0 is considered implemented when:
- all required files exist and run
- scanning produces a manifest
- classification assigns stable categories
- dedupe produces groups
- master index builds correctly
- curated indexes build correctly

At that point, the binder subsystem is ready for v2.1 page resolver work.
